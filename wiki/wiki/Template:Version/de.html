Template zur Angabe von Versionen

Dieses Template kann verwendet werden, um die Versionsnummern von Core und Plugin zu einem Wiki-Artikel hinzuzufügen. Die Versionen beziehen sich dabei direkt auf den Stand, auf dem der Wiki-Artikel basiert.

Hintergrund

Das Template bindet automatisch die aktuellen Versionsnummern ein und prüft beim Aufruf, ob die im Artikel hinterlegte Versionsnummer zur aktuellen Version passt. Ist dies nicht der Fall, wird eine Hinweisbox eingeblendet, die auf die eventuelle Veralterung hindeutet.

Verwendung

Um das Template zu verwenden, setze einfach oben auf deinem Wiki-Artikel folgenden Code und ersetze die in Großbuchstaben geschriebenen Texte.

Artikel zu Core-Funktionen 

{{Template:Version/de|core=CORE_VERSION}}

Artikel zu Erweiterungs-Funktionen 

{{Template:Version/de|core=CORE_VERSION|repo=PLUGIN_REPO|name=PLUGIN_NAME|plugin=PLUGIN_VERSION}}
  • CORE_VERSION

Die Version des Cores von EQdkp Plus zu dem du deinen Wiki-Artikel verfasst.

  • PLUGIN_REPO

Der Name des Github Repositorys der Erweiterung.

  • PLUGIN_NAME

Der Name der Erweiterung in Kleinbuchstaben. Dies wird nur für die Anzeige verwendet.

  • PLUGIN_VERSION

Wie bei der Version des EQdkp Plus Cores, nur, dass hier die Version der Erweiterung angegeben wird.

Besondere Verwendung

Es ist auch möglich, dies für Artikel zu verwenden, die für ein komplettes 'point release' gelten. Dazu muss der passende Teil der Versionsnummer durch ein kleines x ersetzt werden. Beispiel: Wenn der Artikel von Core-Version 2.3.0 bis 2.3.999 gültig ist, solltest du als Versionsnummer 2.3.x verwenden.

In diesem Fall wird der Artikel als für eine zukünfitge Version markiert, bis mindestens Version 2.3.0 veröffentlich wurde und als veraltet, sobald 2.4.0 erschienen ist.
