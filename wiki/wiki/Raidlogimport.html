Settings

This section will describe the different settings of the RLI.

General

  • How should the raids be created?
  • raid note of the raids per hour
  • In which XML-Format is the string?
  • Deactivate Adjustments? - This removes partially gain of DKP per member! Everyone gets all or nothing!
  • Itempools can be saved per item and event. - At the item-import-page the itempool can be saved for all displayed items. On the next import of the raid the itempool is automatically selected for that item.
  • Dont show warnings on deletion?
  • Create zones automatically, if they don't exist
  • Create bosses automatically, if they don't exist
  • Calculation of Timedkp - 0: exact calculation per minute ; >0: minutes, after the member gains full dkp of the hour
  • Delimiter between the Strings, which belong to one "event". - some Tracker allow comment for adjustments
  • Time in seconds, the loot belongs to the boss before.
  • Ignore disenchanted and bank loot?
  • Which DKP shall be used?
    • Boss-DKP - gives DKP for each boss (fixed value or hourly value; defaults can be set at boss/zone management)
    • Time-DKP - gives DKP for hourly attendance in a zone (defaults can be set at boss/zone management)
    • Event-DKP - give DKP for attendance in a zone (set during import)
  • Exists an event for each boss?
  • Add autocomplete function to the following fields?

Member Settings

  • Default rank for automatic created members. - select the default rank
  • How should the member-list be displayed? -
    • Multi-Select - use MultiSelect to select the bosses for each character
    • Multiple Checkboxes - select a checkbox for each raid to select the attendance - very easy
    • Detailed Join/Leave Information - use slider to adjust the attendance - not easy, but very flexible
  • Time in seconds a member can miss without it being tracked. - e.g. for disconnects
  • How many % of attendance do a member need to get the particiaption in the raid?
  • Show member rank?

Standby-Settings

  • Shall the standby-members be assigned to a raid?
  • Shall the standby DKP be absolute? - Means: standby chars get a fixed DKP value, otherwise they can get x % of the DKP
  • Shall standby-members gain start/end-DKP? - from the Attendance Tab
  • How much percent of the DKP or rather how many DKP absolute, shall the standby-members get? - insert 90 for 90%, or 60 for 60 DKP
  • Note for standby-raid
  • Which DKP shall standby-members get? - select which points standby members should get

Attendance

  • Should an extra raid be created for attendency? - if an own raid should be created for raidbegin/raidend, or the DKP should be added to the first raid (raidbegin bonus) / last raid (raidend bonus)
  • Bonus for attendance during raidbegin - bonus for being there at the raidbegin
  • Bonus for attendance during raidend - bonus for being there at raidend
  • Time in seconds, the invite / end of raid lasts. - to determine the raidbegin/raidend
  • Raid note of the start-attendance-raid
  • Raid note of the end-attendance-raid

Bosses/Zones

The RLI Plugin has its own management for Bosses and Zones. This is important because a mapping is needed from the Strings in a log to the real Boss/Zone/Event. If the appropriate settings are activated, bosses, zones and events are created automatically. Please do not forget to associate the events to the correct MultiDKP-Pool.

Boss

  • String - Exact name of the Boss how it will be in the Raidlog
  • Note - will be written to the raid note
  • Bonus points - points for each kill of this boss
  • Points per hour - points for every hour in this boss

Zone

  • String - Exact name of the Zone how it will be in the Raidlog
  • Note - Event-Name. Do not include Suffixes for Heroic etc.
  • Bonus points -
  • Points per hour - points for every hour in this zone

Import your Log

  • Click on "Insert DKP-String"
  • Select the format of the string, if it is not already correctly selected
  • Paste your string into the textbox
  • Click on "Send"

Bonus for Attendance

If you want to give Bonus DKP for attendance, means if they are on time on raidbegin or still there on raidend, this is the right example for you. Go into the RLI-settings and open the Tab "Attendance".

  • Should an extra raid be created for attendency?

It doesn't really matter - set it up if you like it. But creating an own raid makes the life easier for you increasing/decreasing the attendance value for your attendees. But if you don't like having many raids, then set this option to "No" and the points will be added to the first/last boss or the attendees.

  • Bonus for attendance during raidbegin - bonus for being there at the raidbegin

Insert the DKP which the attendees should get for being there at the raidbegin.

  • Bonus for attendance during raidend - bonus for being there at raidend

Insert the DKP which the attendees should get for being there at the raidend.

  • Bonus for complete attendance (all bosskills)

Here you can insert a bonus that members will get which are present during all bosskills

  • Time in seconds, the invite / end of raid lasts. - to determine the raidbegin/raidend

Often the raidlog starts with the invitation phase, and also ends a time after the ingame-raid has ended. Therefore use this setting to determine when the raid really started, so the bonus points can be given for the people that really have participated.

  • Raid note of the start-attendance-raid

Which text should be added as note to the start-attendance-raid (to indicate that start attendance was given)

  • Raid note of the end-attendance-raid

Which text should be added as note to the end-attendance-raid (to indicate that end attendance was given)
